Hexo搭建博客总结

说实话，这是我第二次尝试hexo的搭建，具体的流程参考了以下的博客：

  • Hexo 博客搭建指南 by limedroid
  • hexo边搭边记 by sunnyxx

有了这些引导和帮助，如今搭建个人博客确实不是什么登天的难事了。接下来写写自己遇到的问题以及解决方案：

时区和语言设置

我设置timezone使用北京时间，但需要写成Asia/Shanghai；language比如中文简体就是zh-Hans，在NexT主题中可以参考这里；另外前面都需要加空格。

多说评论（官方宣布2017年6月关闭）

在配置多说评论的时候，由于我自己使用了next主题，那么配置过程只需要在next主题文件目录下找到_config.yml以后，用文本编辑器打开，查找duoshuo_shortname关键字，去掉前面的#注释，在冒号后面空出一个英文空格，接着输入我在多说创建的站点名称，就一切都搞定（比如在多说创建的站点是jack.duoshuo.com，那么shortname就是jack）。由于我使用新浪微博的账号登录多说，因此评论的时候就显示自己的微博账户。如果其他主题的话，也许需要添加额外的js代码。

Disqus评论

设置多说 DISQUS

Github域名

使用Github提供的Github Page服务，需要遵守username.github.io的域名规则，不然访问github.io的时候，就出现了404错误网页，因此为了减少麻烦，建议使用自己在github注册的用户名作为访问github.io的二级域名。

图片资源管理

在管理文章图片资源的时候，可以参考hexo官方的指南（点击中文翻译），通过创建与文章同名的文件夹存放图片不失为一个好方法。

部署错误处理

在部署的时候，我这里出现了Not a git repository (or any of the parent directories): .git的错误，看了别人的解决方案，在自己本地的username.github.io主目录下找到隐藏文件.deploy_git，将其删除后重新执行hexo deploy即可。

避免首页显示全部文章内容

每当next主题展示首页的时候，假如每页显示10篇文章，由于每篇文章在首页默认是全部加载，所以严重影响了首页加载速度。于是可以使用<!-- more -->的分割线安插在文章内容里，从而实现在首页只显示摘要部分，具体参见这里。

添加版权信息

给next主题添加文章版权信息，点击这里，注意事项：

  • 在terminal中进入hexo–>themes–>next–>scripts，然后vim copyright.js（使用vim创建的文本应该是UTF-8编码格式吧），将链接文章中的js代码块copy于此，保存。
  • 回到hexo的根目录，同样vim tail.md，然后使用Markdown写明版权信息，保存。
  • 在hexo根目录下的_config.yml中，找到url: http://yoursite.com的地方，将这里的地址进行替换，替换后的site会作为永久链接中的site

添加站长统计、文章结束、二维码等等

参考博客

hexo本地调试

每次写完文章后，可以首先通过hexo clean清理数据，然后输入hexo g来本地生成静态网页，再通过hexo s启动服务器，接着打开网页浏览器输入http://localhost:4000即可预览效果，等调试没问题后再使用ctrl + c停止服务器，最后hexo d发布。
