计算器实现原理

作业解析

  1. 避免重复小数点
  2. 添加一元运算符：如sin, cos, 还有π
  3. 添加清零按钮C
  4. 添加撤销按钮backspace
  5. 添加“＝”
  6. 添加正负转换按钮

ps：关于显示用户计算步骤的实现，将放在下篇blog中。这里也跳过auto-layout部分，跳过将displayValue设置成optional类型。

避免重复小数点

在计算器中对小数点的处理就是

  1. 只能显示一个小数点，不可以出现3.14..456.78
  2. 如果屏幕显示0，添加小数点后必须是0. 即不能够去除开头零

解决：

  1. 在view controller中加入一个小数点button
  2. 创建一个IBAction func appendDot()作为添加小数点的方法
  3. 声明一个属性 var isDotTyped = false作为判定是否添加了小数点
  4. 实现appendDot()方法

其中isMiddleOfTyping属性是用来除去开头零的，总体思路是：如果没有添加过小数点的话，就在display.text末尾添加小数点，然后将isDotTyped设为true，这样保证了仅添加一次小数点，当然还要再每次op入栈后将isDotTyped设回false，保证下次继续用。如果用户正在middleOfTyoing的时候，就直接在末尾添加小数点即可；如果屏幕上只有0的话，就添加为“0.”然后将isMiddleOfTyping设为true，从而避免去除开头零。

添加一元运算符：如sin, cos, 还有π

  1. 添加sin和cos运算的方法和演示中的“开根”运算一样，以sin为例，首先创建一个Op类型，因为是一元运算，所以创建operation = Op.UnaryOperation(“sin”, sin)第二个sin参数实际上是swift本身定义好的用来实现sin()的方法，由于swift的语言特性，可以直接将方法作为参数传入enum，非常简洁。
  2. 将sin运算类型加入到knownOps字典中即可knownOps["sin"] = operation
  3. 关于π的处理，我就把他作为常数。在后续的作业2中推荐处理为常数，因此需要给Op类型添加新的常数类型，等下次再写。

添加清零按钮C

这里的清零表示清除屏幕的结果，以及运算栈里的所有的op，也就是清空opStack数组，方法是给CalculatorBrain添加一个API － clean()然后实现它，再通过brain.clean()在view controller中调用。虽然容易，不过需要注意还要重置isMiddleOfTyping, isDotTyped等，在作业2中还需要清除variableValues数组

添加撤销按钮backspace

比如实现撤销”345”为“34”，再撤销就是“3”，再撤销就是“0”

  1. 简单的字符串处理，通过NSString的substringToIndex(text.length -1)的方法逐字缩减
  2. 考虑缩减到text.length为0的情况，即空字符串，那就用0补上，避免给用户一个空白，然后就不能再减了。

注意：

  1. label.text 作为String? 不具有substringToIndex()，需要！解包后转为NSString并且只有NSString带有length属性，而String是Struct，没有length属性
  2. 本次作业中也推荐使用countElements() 和 dropLast() 方法，前者获取text的个数，后者返回一个去除末尾字符的string，比上述的方法简洁多了。如果你查看文档中对这两个方法所需的参数类型和返回类型感到困惑的话，那么简单的理解是：String实际上是一个字符串的集合，并且其中的每个字符串都可以用index找到，而且这个集合也能被分割成一个子集合，相当于substring，因此countElements()正好可以传入一个集合，而dropLast()可以传入一个可分割的thing，然后返回一个除去末尾的sub-thing

添加“＝”

注意这里不是在UI上添加“＝”按钮，而是在display上显示 “=” 。当用户点击点击回车按钮（入栈）时，在view controller 的每个displayValue = xxx之后，加一行

即在屏幕更新出结果后，添加一个等号。

实现了输出表达式后就会出现”3 = 3.0”，因为3表示入栈的运算数或运算符，3.0表示运算结果。这样的好处是如果输出多个运算表达式的话就会显示”1+2, 3+4 = 7”，表示最近的一次运算结果为7，然后再入栈一个乘号，就会显示”(1+2)x(3+4) = 21”，不错吧，只是下次再说括号的实现。

添加正负转换按钮

比如“5”转变为“-5”，我以前都是用处理字符串的思维解决正负号问题，像 if text.hasPrefix(“-”)之类的方法。现在看了作业提示后发现，需要将正负转换操作视为一个一元运算，仔细想想看是有道理的：1.在后期的作业中添加了undo()就可以支持撤销的操作；2.所有的一元运算支持括号保护，比如“3+(-5)”

思路：

  1. 如果用户正在isMiddleOfTyping，那就在text前面加上或取消“－”，然后修改enter()方法，在brain.pushOperand()之前检查这个operand前面有没有负号，如果有的话，就取消负号，然后push到opStack中，然后再往stack里加入一个正负转换运算。如果你要问为什么这么麻烦？其实只是为了括号保护，具体在作业2中会有解答，因此就可以输出“3+(-5)”
  2. 如果用户已经把operand推入stack之后，再去转换它的正负值的话，就把该操作等同于其他一元运算的方法即可。

解决：

  • 创建正负转换运算，并加入knownOps

特别注意：这里不是数学减号，你可以理解为负号。而且减号与负号都不可以用键盘上的“－”表示。在demo中，Paul选择xcode menu -> Edit -> Special Characters，里面找到math symbols，然后在窗口中，你可以看到第一个是数学加号，第二个是MINUS SIGN作为减号，第三个是HYPHEN-MINUS，相对短一些，你可以将它设置为这里的负号。如果你说这么麻烦，干嘛不定义个像sin，cos一样的名字，比如“rev”？那么结果就是：计算结果不受影响，但是输出表达式就成了”3+(rev5)”。所以为了能够输出正确的表达式，所有的数学符号一律需要从那个Special Characters中获得。
