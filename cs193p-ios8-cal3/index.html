斯坦福CS193p-iOS8课程作业之计算器3

本篇是计算器作业系列的第三章。这里需要实现的就是：在之前做好的计算器中输入一个一元运算函数的表达式，然后绘制成为函数图形。

我们可以假象现在有个“y = 2x+1”的表达式，那么如何绘制这个函数呢？

环境：iOS 8, Swift
作业：Programming_ Project 3.pdf (Assignment 3)
资源：点击进入

确定思路

记得上初中几何课的时候，我经常拿着铅笔尺子圆规做图。比如画函数y = 2x+1，那么先绘出x轴和y轴，标好单位。因为这个函数是一条直线，那么取两个点就可以定位了：当x = 0时y = 1，当x = -0.5时y = 0，然后连接点(-0.5,0)和(0,1)，就画出了函数图。

现在这里的要求是用程序来做函数图，对于没有尝试过的程序员来说，可能一开始不知道怎么下手，像我一样。其实，根本不需要把过程想的多么复杂，本质就是上几何课时候的画图技巧。在UIView中，绘制一个一元函数就是绘制一连串的CGPoint(x, y)，其中x与y的关系可以通过函数表达式来确立，从屏幕最左端x = 0开始，找到对应y的取值，然后一直到屏幕右端（比如x = 340），遍历每一个x，并找到对应y的取值，组成CGPoint，再把每个点连成线，就完成了绘制。需要注意的是：UIView的坐标和画几何图的坐标位置不同，需要进行一个转换，UIView的坐标原点在左上角，而我们绘制的坐标轴位于屏幕正中心。所以这里需要建立一个转换，实现坐标原点的平移和y轴的反转，即从屏幕最左侧到最右侧的取值(0<px<340)，需要转换为函数中的x取值，比如(-3<fx<3)，然后把函数x值带入函数y = 2x+1获取y的值，再将y转换为屏幕上的py；由于不同屏幕的分辨率不同， 为了兼顾Retina屏幕，需要遍历每个pixel，而非CGPoint

展示函数表达式

GraphViewController是由点击计算器的一个按钮Graph而出现并显示函数图像，是一个新的MVC，那么他不仅要绘制函数，同时需要在屏幕某个地方展示函数的表达式，以便用户知道当前绘制的是什么函数。

需要解决的问题就是，如何将计算器中Stack里的各种运算数和运算符转化为可储存和解析的类型呢(Property List)？之前的计算器中，stack里面储存的是Op类型，op是enum，并在之前的项目中实现了Printable协议，因此可以转换为string，并且CalculatorBrain中添加了一个program: AnyObject，这个就是将opStack转化为property list的数据结构。具体的实现方法在视频中已经给出了方案(视频5. Objective-C Compatibility, Property List, Views.mp4)。这个封装好的program就可以发送给其他类使用，不过如何能把他解析成表达式呢？可以建议在CalculatorBrain中增加一个方法（也可以是类方法）

通过调用之前写好的方法，将program解析称为表达式，然后在GraphViewController中接收解析的结果，比如sin(m)， 然后在前面加上“y = ”就可以在屏幕上显示一个完整的函数表达式；需要特别注意的是，如果解析的表达式结果为“sin(m), m+2”，说明此时计算器的stack里面有两个独立的表达式，那么绘制的函数自然是最近期输入的表达式（即后者），但是不能给用户展示“y = sin(m), m+2”，需要改为”y = m+2”，如果用户再次输入乘号，就显示“y = sin(m)x(m+2)”并绘出函数。这里可以运用componentsSeparatedByString()把NSString以逗号为分隔符，分割成数组，然后返回最后的那个。
