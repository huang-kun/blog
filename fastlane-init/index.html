fastlane初始化

fastlane的初始化需要依赖一个项目，首先进入terminal，将路径设置到自己项目的主目录中，然后调用fastlane init，接着命令行会向你索取苹果开发者的id，输入后即可自动被写入fastlane的配置中，顺利的话，最终在项目主目录下生成一个名为fastlane的文件夹。

进去后，发现里面存在一些文件：

  • Appfile
  • Deliverfile
  • Fastfile
  • metadata（文件夹）
  • screenshots（文件夹）

前面三个是默认的配置文件，其中Appfile存放着项目的app_identifier, apple_id, team_id等基本信息

Deliverfile是fastlane执行fastlane deliver命令时需要的配置参数文件，主要是用于上传iTunes Connect的一些基本配置

metadata

这里是储存和管理项目元数据信息的文件夹，比如包含了项目的：

  • 图标 (AppIcon.png)
  • 版权 (copyright.txt)
  • 本地化信息 (en-US, zh-Hans)，包括更新说明、商店搜索关键字、相关网站等
  • 分类信息 (比如游戏类、即时战略游戏等)
  • 审核信息等

screenshots

这里用来储存app在商店中展示的所有截图，里面包含了：

  • README.txt
  • screenshots.html（可能没有）
  • 本地化文件夹，即不同语言文件夹会存放自己相应的所有截图

其中screenshots.html是截图预览的本地网页文件，双击以后可以直接查看所有设备截图的预览，十分方便。

而README.txt里的说明也很给力，即所有的截图可以随便命名，因为只要图片本身的像素尺寸准确无误，那么该图片就会自动分类到相应的设备中去。比如该图片尺寸为750x1334，那么就可以自动归位至iPhone 6或7的4.7英寸设备屏幕中。这样就满足了即使不使用fastlane自带的截图生成服务，依然可以直接把设计师做好的截图直接拿过来用，并且避免了图片命名和手动上传截图的枯燥工作。（附：来自PaintCode官方的iPhone分辨率终极指南）

其他说明

在新老项目中分别使用fastlane

如果从Xcode新建一个工程，然后fastlane init，那么初始化的时候，fastlane也会顺便在iTunes Connect中生成相应的app，并且在两边都设置好初始的信息。

如果在老项目中fastlane init，并且在iTunes Connect已经提交给这个app的话，那么fastlane会自动获取它的图标、所有提交的描述信息，以及所有截图全部下载到fastlane文件夹中，以保持同步。

初始化出错

如果执行fastlane init过程中出现异常，那么fastlane会自动删除原本生成的fastlane文件夹，然后将metadata和screenshots文件夹直接放在项目的主目录下，所以除了项目目录中多了两个文件夹以外，基本什么也没有发生过。至于出错的原因，很可能是大环境问题，建议更新ruby和fastlane版本，在我写的fastlane安装博客中正好遇上了不少坑，有可能会帮助到你。

目录问题

在官方文档中，关于目录有以下明确的解释：

Notice how every action and every plugin’s code runs in the root of the project, while all user code from the Fastfile runs inside the ./fastlane directory.

我翻译成贴切的用语是：

  1. 在项目主目录下运行各种lane就好，不要在./fastlane的目录下运行它们
  2. 在配置文件或者路径的时候（比如Gymfile），记住是从主目录去调用的，确保使用的文件路径正确

如果当你遇到以下情况：

  1. 通过terminalcd到项目主目录下，运行fastlane xxx
  2. 需要修改Fastfile，接着cd到fastlane目录下进行文件修改和保存
  3. 然后为了测试修改效果，又要返回到主目录下，再运行fastlane
  4. 反复切换目录，WTF…

这么做虽然正确，但是操作太过于繁琐。有个解决方法是：在terminal中开启两个tab，其中一个tab专门定为到./fastlane目录，用来修改Fastfile以及各种file；另一个tab专门定为在主目录，进行各种fastlane命令。只需要切换tab就好了。

通过Fastfile运行shell或脚本

如果在项目的主目录下有个名为hello.txt文件，那么使用命令行打开该文件的话，可以cd到项目路径，然后open hello.txt，当然该操作可以放在Fastfile中，通过sh "..."的格式来执行。

这样设置Fastfile之后会发现，无论在项目主目录下还是在fastlane目录下运行fastlane hello，都会出现No such file or directory的错误。执行lane的时候目录都会默认在./fastlane，该目录下是不存在hello.txt文件的。既然如此，就可以修改命令为sh "open ../hello.txt"即可生效；或者将hello.txt文件转移至./fastlane目录下，进行sh "open hello.txt"也可以生效。

同样，在lane中运行自己写的脚本也是如此，比如sh "my_script.sh"，关键确保脚本的位置和lane中设置的位置正确，才可以顺利运行。
