fastlane的简介和安装

fastlane简介

fastlane是一款开源的ruby脚本工具，用于app进行自动化测试、打包、提交、发布的一体化服务，支持iOS和Android

fastlane的优势体验在哪

以前打包iOS的app需要Xcode，提交app需要打开浏览器进入iTunes Connect，如今fastlane使用命令行就能搞定一切，而且效率翻倍，因为省去了加载iTunes Connect网页必须的js和css等元素。除此之外fastlane还可以：

  • 自动生成证书和provisioning profile
  • app快照截取
  • TestFlight或蒲公英等其他测试平台的集成
  • Jenkins整合
  • 自定义和整合以上工作流（即命令行中输入一行命令搞定一切）
  • 使用命令行进行苹果开发者中心和iTunes Connect的常用操作（比如查看和修改app状态，添加测试者和开发设备等）
  • 更多特性详见官方github地址

以下这段摘要来自Felix Krause博客

在过去的8个月里，fastlane为全球开发者节省了100万个小时，相当于114年。（该博客发布于2016年4月）

有趣的是，他们还做了一个叫做fasttrack的网页app来查看fastlane的实时使用情况，当有人启动fastlane服务的时候，就会从屏幕下方发射一个小火箭（其实是小方块）

官方安装流程

如果你还没有安装过fastlane，建议直接跳至下面的：新安装流程

按照官方的安装步骤就可以进行安装，但是当我第一次上手以后，就踩了不少坑，具体问题如下：

  • 因为图方便，在Github的fastlane首页里下载了zip包，然后解压安装
  • 使用fastlane init以后出错，显示为undefined method edit_version for nil:NilClass等一系列提示，然后将生成的fastlane文件夹又给删除了；上网查找原因后得知官方已经修复该问题，需要至少2.28.1版本的支持，相关网址
  • 于是我去升级fastlane，使用Rubygems的途径能获得指定版本，升级后发现在命令行中的fastlane竟然还是老版本？！于是上网查找原因后，得知通过不同的安装途径会得到多个fastlane的版本安装在了不同的位置，结果系统始终默认使用的是我通过zip包安装的版本，所以即使通过Rubygems升级，得到的新版本也会被自动忽视，于是解决方案是：首先删除通过zip包安装的fastlane，然后使用fastlane的话，系统就默认为后者了，相关网址
  • 通过fastlane进行sigh操作来管理pp文件的时候，出错提示Connection reset by peer - SSL_connect，官方解释说系统的ruby为2.0.0版本太低，升级后就好了。结果使用brew升级后发现命令行里还是老版本ruby，所以需要将系统版的ruby替换为brew版本的ruby，方法在新安装流程中介绍。相关网址

新 ∙ 安装流程

踩了坑以后，我还是总结一个新的安装流程：

  1. 首先安装最新的ruby，然后替换掉mac系统的旧版本ruby
  2. 统一通过一种途径进行fastlane的安装和升级，且安装版本不低于2.28.1（目前最新是2.28.7）

安装Ruby

首先打开terminal，查看下默认的Ruby的版本和路径，如果版本大于2.3.1的话，应该不成问题。

安装fastlane

在macOS平台上，官方提供了三个途径进行安装：

  • homebrew: brew cask install fastlane
  • 下载安装包（不建议）
  • Rubygems: sudo gem install fastlane -NV

接下来是三选一。我这里使用了Rubygems的途径，并且将之前选择了其他途径安装的fastlane，一律删除。注意如果安装了fastlane以后才升级的Ruby，那么Ruby升级成功后，依然需要重新安装fastlane

至此，安装搞定，之后可以看官方文档和其他教程了。但愿该文章有助于新手避免踩坑，或者向已踩过坑的开发者分享一些解决方案。Raywenderlich的这篇fastlane Tutorial文章已经是个很不错的入门指导，文章里覆盖了使用fastlane的基本特性：

  • 初始化
  • 证书管理
  • fastlane目录结构
  • app信息管理
  • 快照截取
  • 打包
  • 上传iTunes Connect
  • 提交审核
