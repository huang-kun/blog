iOS开发者证书的申请到安装（多图）

作为一个开发者来说，看着自己写的app能首次运行在自己的设备中，那种小兴奋的感觉是难以忘却的。具体怎么实现呢？虽然现在已经不是第一次真机测试app，不过当证书到期需要更新的时候，仍然要了解这些知识，所以我就索性总结一下吧。

真机运行的前提条件：需要购买苹果99美元一年的iOS开发者项目，拥有自己的苹果开发者账号（developer.apple.com）。

说明：这里展示的操作系统是英文版的Mac OS X Yosemite

整个过程需要经历五个阶段：

  1. 生成证书请求文件（本地）
  2. 创建证书（网络）
  3. 创建App ID（网络）
  4. 添加设备（网络）
  5. 创建和安装Provisioning Profile（网络＋本地）

生成证书请求文件

在mac中打开‘Keychain Access’

在菜单中选择’Certificate Assistant’->‘Request a Certificate from a Certificate Authority’

在这里需要输入邮箱和名字，然后必须选择“Saved to disk”！

最终会在你选择的路径中生成一个名为“CertificateSigningRequest.certSigningRequest”文件，到这里结束第一阶段工作。

创建证书

登录developer.apple.com，选择‘Certificates, Identifiers & Profiles’

再选择’Certificate’->‘Development’

注意：每位开发者只能拥有一个开发者证书（看清Type那里是iOS Development，不是别的），如果之前你已经创建过了的话，那么需要吊销旧的证书后才能创建一个新的证书，所以需要你自行决定是否保留或者吊销原来的开发者证书。因为我原来已经给自己创建了开发者证书，现在我选择吊销旧的证书，点击”Revoke”即可。

现在我自己没有其他开发者证书了，点击右上角的“＋”按钮，开始创建一个新的开发者证书。

因为我这里只需要开发者的证书，所以选择“iOS App Development”

接下来的页面是告诉你如何在本地创建一个请求文件，教你如何从Keychain Access生成的一个请求文件，这就是一开始的那个步骤，所以我已经完成了。

然后进入到这里，选择“choose file”，上传一开始在本地生成的那个CertificateSigningRequest.certSigningRequest文件

点击“Generate”后等待一会儿，接着就成功创建了一个开发者证书。

点击“Download”，下载成功后，我直接从Safari的右上角找到下载的.cer文件，双击打开（也可以从下载的disk里双击打开）

最终在Keychain Access中出现一个证书，附带一个私有的钥匙。结束第二阶段工作。

创建App ID

回到之前的网页，选择“App ID”

点击“＋”

这里到了注册App ID的界面，一个完整的App ID是由两部分组成：App ID Prefix(前缀)和App ID Suffix(后缀)。

前缀的信息是默认生成的字符串，代表着“Team ID”，顺便在下面写个id的名字作为描述。

后缀的信息需要你来做选择

Explicit App ID就是给针对性的app创建的id，比如你的app需要支持游戏中心、内购、iCloud等特殊需求，或者你只想让你的开发者证书绑定某一个App而不是所有的App，那就必须选择Explicit App ID；

Wildcard App ID就是如果你没有上述那么多特殊要求，或者你希望使用这个id能开发并且真机测试多个app，而不只是绑定一个app开发的话（尤其是学习iOS开发的情形），那就选择它吧。

在下面的App Services中，可以自行勾选你所需要的服务，其中灰色的部分不可选是因为我在之前选择了后缀为Wildcard App ID的缘故。

提交完成后就生成了iOS App ID，第三阶段结束。

添加真机设备

比如在你的iPhone, iPad等设备上运行你自己做的app，就需要添加设备。

在网页中选择“Devices”-“All”

点击“＋”添加设备

在这里输入你的设备名称和udid，但什么是udid呢？下面解释到这是你的设备的“身份证”，跟公民的身份证具有一样的唯一性和身份识别的功能。但是如何获得你的iPhone的身份证呢？

将你的iPhone连接到mac，打开iTunes(演示版本：12.1.0.50)，然后在Summary中可以看见自己的iPhone的信息，可惜还是找不到udid，奇怪了，下面只有Capacity, Phone Number, Serial Number这些信息。接着神奇的事情出现了，用鼠标点击Serial Number，就出现了udid替换了原来的信息，原来是隐藏了“身份证”！(点击Phone Number和Serial Number可以被替换为其他的信息)

这么长的字符串，估计不会有人真愿意手动输入吧！还好可以右键复制，然后粘贴到网页中去，按照指示继续完成设备添加就行。

添加成功后你就可以看到自己的设备显示在网页中（下面还说你还可以继续添加99个设备），那么第四阶段完成。

创建和安装Provisioning Profiles

选择Development

这个就是最终让你开发的app能够安装并运行在你的设备上的东西，之前创建的所有的零件就是为了组装成这个Provisioning Profiles，到这里还是点击“＋”

接下来是本地安装的部分：下载后建议不要双击打开，否则的话就会生成类似28CED985-346e-2537-9C2A-8923A4A678F57.mobileprovision的文件名，如果你希望看到的是显示你在网页中自己取的文件名字的话，推荐以下两个步骤：

方法1. 把下载好的profile文件直接拖拽到iTunes中去，然后什么反应也没有，然后就已经神奇地安装好了！

方法2. 选择Library -> MobileDevice -> Provisioning Profiles

如果找不到Library文件夹的话，就在finder中按下Command+Shift+G，输入以下路径，一步到位。

然后把profile文件直接拖拽进来就好。由于我是按照方法1直接拖到iTunes中的，所以当我进来的时候，就看到自己的profile文件已经在自动在这里安置好了，而且关键是名字仍然是我自己起的名字，没有被解码成下面那些文件名，这样不是看着很方便吗？

测试：

创建Xcode项目，在Organization Identifier中设定正确的信息

如果是测试运行原先的app而非新建的项目，那么需要打开原先项目的info.plist，确保其中的Bundle identifier与新注册的一致

最后的工序，即设置Code Signing，即Project -> Build Settings -> 搜索code sign，其中选项的部分Xcode会自动找到Keychain Access中匹配的证书，全部选择为刚申请的证书。

连接并选择你的iPhone作为运行设备

写好代码，运行，如果没有意外的话，你会在自己的iPhone中运行HelloWorld!（推荐做一个带有内容为Hello World的label，显示在你的iPhone上，总比直接在控制台打印Hello World的效果要好多了吧）

流程到此结束。

注意问题

如果遇到Build failed - 显示Couldn’t code sign问题，说明还是证书的配置的问题。虽然自己明明申请的证书，为什么会出错呢？通过研究后发现是因为我的Keychain Access本身存在之前的旧证书，所以加入的新的证书后难免会造成混淆，因此建议把旧的、过期的、不用的证书从本地到网络都统统清理干净后，问题就解决了，因此保持一个简洁的环境有助于更好地进行文件管理和减少问题的出现。

查看你的证书情况

下载的证书建议备份起来，如果因为系统内的证书丢失而造成的不便，是得不偿失的。在Xcode中通过选择Preferences -> Accounts -> 选择用户 -> 右下角的View Details，就可以看到Xcode自动安装的Provisioning Profiles，因为大多数情况开发者安装的Profile文件都是双击后安装的，所以从文件夹中不好分清哪个是哪个profile文件，不过可以从这里进行大致的浏览。

如果你的iPhone设备连接上了mac，可以通过Xcode菜单 -> Window -> Devices -> 选择左侧的设备 -> 选择左下方的设置 -> show provisioning profiles 查看该设备已经存在的profile文件

并且可以通过底部的“＋”或“－”进行添加或删除。
