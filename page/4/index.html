开发SET卡牌游戏技术总结

自从我发布了这个JKSET作为开源项目，我认为应该写点什么，帮助有兴趣了解这个游戏或者代码的朋友们。点击进入源码地址

游戏简介

这是一款源自斯坦福iOS 7开发课程的第一次大作业项目，项目背景是一款曾经风靡20世纪90年代歪果人的游戏，名叫SET。Set游戏是由一位名叫Marsha Falco的女士于1974年设计的一款卡片游戏，并且由Set公司在1990年正式推出。Marsha曾是一名专注于癫痫是否遗传于德国牧羊人问题的人口遗传学家。为了研究犬类细胞的基因和染色体，Marsha为每只测试犬创建了文档卡片以记录信息，不同的是，她通过绘制图案的方式来记录数据，使用不同的图案及不同的属性来代表不同的基因组合。后来当她在与其工作的兽医解释这些组合图案的时候，她认为还可以给这些图案组合加入更多有趣的东西，之后就诞生了SET游戏。几年后，Marsha通过与家人朋友的游戏过程中对其重新进行了完善，最终于1990年发布了这款游戏。

游戏中的每张卡片中都含有4种元素，每种元素包含3种选择 － 数量（1个，2个，3个），图案（钻石、曲状物、柱形），效果（实心、条状、空心），颜色（红色、绿色、紫色）。一副完整的卡牌共有81张卡片，而玩家的任务在于每次选出3张卡片来组成一对SET，并且从81张卡片中挑选出所有可能的SET。

组成一对SET的唯一条件是：所选3张卡片中的每个元素，只需满足－要么“都相同”，要么“都不同”。

iOS手势交互

需要知道的事

  • 在view添加的gesture中，可以通过这个gesture.view找到相关的view
  • UIImageView是仅有的几个UIKit子类中的特例，因为他默认userInteractionEnabled = NO，所以在它上面应用手势识别之前需要确保修改这个默认。
  • 通过locationInView:的方法可以得到从gesture探测到的手指触屏坐标，但如果是多个手指触到的屏幕呢？那么返回的坐标点是这几个手指的中心。比如两个手指的话，就是两个指尖距离的中点。或者可以通过locationOfTouch:inView:来获得具体每根手指的触屏坐标。
  • 对于tap, swipe等手势的状态只能响应StateRecognized，而不会响应StateBegan, Changed, Ended.
  • 关于StateCancel的状态是由于系统放弃对目前手势的识别，具体情况有：突然来电话，或当设备靠近脸部的时候等。因此要处理好Cancel或Fail的状态。
  • pinch和rotation手势是根据view的center作为轴心进行缩放或旋转，而不会把两个手指间的距离中点作为轴心。但是也可以修改的，view.layer有个anchorPoint的属性基本上代表了view的轴心，而两手指间的中点可以通过locationInView:获得，这样就可以通过修改view的轴心为两手指的中点从而改变效果。

斯坦福CS193p作业之打砖块

Resource: Stanford cs193p 2015 winner (iOS 8)
Assignment: Breakout(打砖块)
资源：vojtamolda/cs193p （这是别人的源码，我自己写的已经不维护了）

作业要求

  1. 运用UIDynamicAnimator创建Breakout游戏，但是不要一模一样地模仿示例UI
  2. 当击中砖块的时候，需要执行一些动画，比如砖块在消失之前进行翻转或者变了颜色，主要是练习UIView的animation
  3. 添加一个pan手势用来控制板子，还需要支持tap手势，在随机的位置发球
  4. 当游戏结束时候，弹出一个alert，并且重设游戏
  5. 你的游戏需要设计至少4个不同的变量来控制游戏的细节，比如砖块数量、弹力、球的数量、带有特殊行为的砖块等
  6. 使用一个tabBarController，添加一个tab，目的是让用户来设置游戏至少4个变量的具体信息，设置完毕后，游戏应该立即作出反应。
  7. 在这个实现设置功能的MVC中至少使用以下之一：UISwitch, UISegmentedControl, UIStepper/UISlider
  8. 你的游戏参数配置必须在app运行期间持续存在
  9. 同时支持iPhone和iPad

斯坦福CS193p作业之Smashtag

环境：iOS 8, Swift
作业：Programming_ Project 4.pdf (Assignment 4)
资源：vojtamolda/cs193p

作业要求

  1. 用不同颜色来标记不同的标签、url和用户名（凡是具有提示性的信息）。 // [IndexedKeyword]
  2. 当用户点击其中一个tweet的时候，segue到一个新的UITableViewController（以下我就称其为MentionsTableViewController），其中包含4个section来展示所有的提示性信息：比如图片、url、标签和用户名。图片要求是展现在首个section，并且每行只显示一个图片，其余三个就放在后面的三个section里面，也是每行展示一个信息。// tweet.media.url
  3. 上述的每个section都需要一个合适的header
  4. 如果该section没有信息可展示的话，那么就不需要header
  5. 如果用户点击标点或者用户名，就segue到一个新的vc显示搜索该标签或者该用户名的结果，注意搜索的标签是要带有前缀符号的，例如“#stanford”而不是“stanford”，搜索“@Jack”而不是“Jack”，而sugue到新的搜索结果展示vc必须与TweetTableViewController的展示效果一致。
  6. 如果用户在MentionsTableViewController中点击url的话，要求用Safari打开url
  7. 如果用户在MentionsTableViewController中点击一张图片的话，segue到一个新的专门可以放大缩小滚动图片的MVC，当首次加载该图片的时候，要求展示图片的原比例大小，而且周围没有“白边”。
  8. 在app中展示最新的100个twitter检索结果，加入一个UITabBarController，展示两个tap，一个是用来显示可以搜索的tableViewController，另一个用来显示最新搜索结果的tableViewController，对于最新的检索结果展示不能有重复且最近更新的展示在最上面。用户点击结果的话，就用segue展示相关的tweets。用NSUserDefaults储存最新的检索信息，这样的话当重启app的时候就可以直接显示了。
  9. 绝对不可以因为任何网络的请求而导致主线程受阻。
  10. 这是个iPhone-only的app，还要保证在portrait或者landscape都能正确展示信息。
